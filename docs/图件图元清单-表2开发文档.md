## 图2.1 系统总体架构图
目的：展示端到端组件与通信。

图元清单：
1. Client Zone：Electron Client、Browser Extension。
2. Presentation Zone：Frontend (Vue3)。
3. Governance Zone：Backend (Spring Boot)。
4. AI Zone：Python AI Inference。
5. Data Zone：MySQL。
6. Evidence Zone：Evidence Anchor (External Time Source + RSA Sign)。

连线：
1. Client -> Backend: /api/client/report (HTTPS)。
2. Frontend -> Backend: /api/* (JWT)。
3. Backend -> Python: Feign/HTTP internal (/predict/risk, /train/*, /api/adversarial/*)。
4. Backend <-> MySQL: governance_event, approval_request, audit_log, hash_chain。
5. Backend -> Evidence Anchor: payload_hash anchor + verify。

标注：
1. 在 Backend 外围加“company_id scope”虚线框。
2. 在 Frontend->Python 画红色叉，标注“禁止前端直连AI服务”。

## 图3.2 全链路时序图
参与者：Client, Gateway(Backend), PolicyEngine, ApprovalService, AuditService, AnchorService。

时序消息：
1. Client -> Gateway: report()
2. Gateway -> PolicyEngine: matchRules()
3. PolicyEngine -> Gateway: alert
4. Gateway -> ApprovalService: createRequest()
5. ApprovalService -> Gateway: approved/rejected
6. Gateway -> AuditService: writeAudit()
7. AuditService -> AnchorService: anchor(payload_hash)
8. Gateway -> Frontend: status update

关键ID标注：event_id、request_id、current_hash、anchor_id。
